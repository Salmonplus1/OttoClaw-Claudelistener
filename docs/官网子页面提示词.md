## OttoClaw 项目概述

**名称：** OttoClaw（小猫一号 · 静默版）

**定位：** 一个不会开口说话的 AI 桌面人形机器人 — ESP32-S3 单芯片全栈运行，专为 i 人设计。

**GitHub：** https://github.com/FlashCat-Jordan/OttoClaw

**官网页面应链接到：** https://www.shanmaotech.cn/ottodiy

**许可证：** CC BY-NC-SA 4.0（全栈开源）

**核心卖点（6 个）：**

### 1. 真正运行在本地端侧的轻量 Agent
- 纯 C / FreeRTOS，单块 ESP32-S3 即可运行全部功能
- 不依赖云端服务器，记忆、会话、技能全部本地存储
- 0.5W 功耗 24/7 在线
- 无 Linux、无 Node.js、无臃肿依赖

### 2. 不会开口说话 — 专为 i 人设计
- 不像其他机器人那样跟你说话打扰你
- 采用钉钉消息交互，忙碌时它安静等候，空闲时看一眼消息即可触发响应
- 安静如猫，始终在场
- Captive Portal 配网：连接热点自动跳转配置页面

### 3. AI 自主情绪表达 — 22 种情绪随语境波动
- 22 种情绪状态：开心、害羞、思考、愤怒、惊讶、无聊、赛博、发晕、亢奋...
- 情绪是 AI 自主触发，不是用户命令触发
- 开心时摇摆，害羞时低头掩面，思考时做出沉思姿态
- LCD 屏幕实时显示当前情绪状态

### 4. 性格与成长体系 — 从高冷到羁绊
- 5 阶段关系成长：陌生(0~79) → 认识(80~199) → 熟络(200~499) → 亲密(500~2999) → 羁绊(3000+)
- 初次见面爱答不理，聊多了自然熟络
- 关系类型由 AI 自主推演：兄弟、恋人、铁哥们...
- LCD 右上角红色爱心 1~5 颗，关系升温的见证
- 用户 rude/怼 → 关系扣分

### 5. 真正的 AI 控制每一个关节
- 大模型自主决定 6 个舵机到达何种角度
- 不是预设脚本，是 AI 实时想象出来的动作姿态
- 实现了 AI 意识的物理化表达
- 示例："来一个求婚的动作" → AI 推理：单膝跪地 + 右手高举 → 机器人真的跪下求婚

### 6. 全栈开源 + 全开放架构
- 硬件、软件、3D 模型全部开源（CC BY-NC-SA 4.0）
- 自选模型：Anthropic兼容 + OpenAI兼容，自定义 Base URL 即可接入任意模型
- 自选通道：钉钉 Stream 直连（无需公网服务器）
- 阿里云百炼一键接入：搜索增强、Agent 应用、MCP 服务
- 100+ 社区二次创作

## 烧录方式

**方式一（最简单，推荐新手）：**
1. 从 GitHub Releases 下载固件 ottoclaw-full-v2.0.bin
2. USB 连接开发板，保持开机
3. 打开 https://www.16302.com/localinit（鹿戴马在线烧录）
4. 芯片选 ESP系列 > 不限类型，上传固件文件（写入地址 0x0）
5. 点击"开始烧录"，完成后按 EN 键重启

**方式二：esptool 命令行**
```bash
esptool.py --chip esp32s3 --port PORT --baud 460800 --before default_reset --after hard_reset write_flash -z 0x0 ottoclaw-full-v2.0.bin
```

## 技术架构要点（可选放页面底部）

- 纯 C / FreeRTOS — 单块 ESP32-S3 运行全部功能
- 双核架构 — Core 0 处理网络 I/O，Core 1 运行 Agent 循环
- Anthropic tool use / ReAct 循环 — AI 自主决定工具调用
- 6 舵机 LEDC PWM — 每个关节独立控制
- SPIFFS 本地存储 — 记忆、会话、配置均在设备本地
